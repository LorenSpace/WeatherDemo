列表


List是连续的容器，而vector是非连续的容器，即list将元素存储在连续的存储器中，而vector存储在不连续的存储器中

正是因为vector不是连续的。所以它在插入和删除时，需要大量时间移动元素

List支持双向，并为插入和删除操作提供了一种有效的方法。在列表中遍历速度很慢，因为列表元素是按顺序访问的，而vector支持随机访问


基类


从一个类派生出另一个类时，原始类称为基类，继承类称为派生类

在初始化构造函数时，除了一般的赋值方法，还可以直接采用基本的列表赋值


#include<iostream>
#include<string>
using namespace std;

class TableTennis{

    private:
        string firstname;
        string lastname;
        bool hastable;
    
    public:
        TableTennis(const string & fn = "none", const string & ln = "none", bool ht = false);

};

TableTennis :: TableTennis(const string & fn, const string & ln, bool ht) : 
firstname(fn), lastname(ln), hastable(ht){

}

//这两种写法一样
// TableTennis :: TableTennis(const string & fn, const string & ln, bool ht){

//     firstname = fn;
//     lastname = ln;
//     hastable = ht;

// }

在c++中，派生（继承）一个类使用：其余和Java大差不差

有关派生类构造函数的要点如下

创建基类对象

派生类构造函数应通过成员初始化列表将基类信息传递给基类构造函数

派生类构造函数应初始化派生类新增的数据成员


#include<iostream>
#include<string>
#include "TableTennis.h"
using namespace std;

class TableTennisVip : TableTennis{

    private:
        int rate;

    public:
        TableTennisVip(const string & fn = "none", const string & ln = "none", bool ht = false, int r =0);
        int Rating() const;

};

TableTennisVip :: TableTennisVip(const string & fn, const string & ln, bool ht, int r) : 
TableTennis(fn, ln, ht){//通过使用列表，来将值传入到基类中

    rate = r;
    
}

如果不使用列表操作，就会默认为基类的默认值值

和直接使用TableTennis()是一样的


TableTennisVip :: TableTennisVip(const string & fn, const string & ln, bool ht, int r) : TableTennis()
TableTennisVip :: TableTennisVip(const string & fn, const string & ln, bool ht, int r)

创建派生类的对象，程序首先会调用基类构造函数，然后再调用派生类构造函数

派生类过期时，程序会先调用派生类的析构函数，然后再使用基类的

基类的指针和引用可以不用显式转化的直接指向派生类的对象或者引用派生类的对象（不可以将基类对象和地址赋值给派生类）


题外话-格式化输出


在之后的内容中会有详细的描述格式化输出问题，南邮的acm题目中也有对应的东西

c++ 使用 cout.setf(ios_base::fixed, ios_base::floatfield) 进行格式化输出

ios_base::fixed表示：用正常的记数方法显示浮点数(与科学计数法相对应)；ios_base::floatfield表示小数点后保留6位小数


fmtflags setf(fmtflage) //第一原型
fmtflags setf(fmtflags, fmtflags)  //第二原型
//fmtflags 是 bitmask 类型(一种用来存储各个位值的类型)的 typedef 名

标志 功能
boolalpha 可以使用单词”true”和”false”进行输入/输出的布尔值
oct 用八进制格式显示数值
dec 用十进制格式显示数值
hex 用十六进制格式显示数值
left 输出调整为左对齐
right 输出调整为右对齐
scientific 用科学记数法显示浮点数
fixed 用正常的记数方法显示浮点数(与科学计数法相对应)
showbase 输出时显示所有数值的基数
showpoint 显示小数点和额外的零，即使不需要
showpos 在非负数值前面显示”＋（正号）”
skipws 当从一个流进行读取时，跳过空白字符(spaces, tabs, newlines)
unitbuf 在每次插入以后，清空缓冲区
internal 将填充字符回到符号和数值之间
uppercase 以大写的形式显示科学记数法中的”e”和十六进制格式的”x”

除此之外，你还可以进行指示符的关闭和开启操作


操作符 描述
resetiosflags(long f) 关闭被指定为f的标志
setiosflags(long f) 启用指定为f的标志

第一原型比较简单且容易理解，可以直接设置cout的输出方式


// 形式一：
// cout << showpos << 255 << endl;

// 形式二：
// showpos(cout);
// cout << 255 << endl;

// 形式三：
cout.setf(ios_base::showpos);
cout << 255 << endl;
//输出: +255

cout << setiosflags(ios_base::scientific) << 255.255 << endl;
//输出： 2.55255e+02
cout << setiosflags(ios_base::scientific); //启用scientific标志
cout << resetiosflags(ios_base::scientific); //关闭scientific标志
cout << setprecision(4) << 255.255 << endl; //设置输出数值的精度为4位
//输出： 255.3

第二原型看似很麻烦，但是只需要记住模板就好，调试时，也会更加方便


第二个参数 第一个参数 含义
ios_base::basefield ios_base::dec 使用10进制
ios_base::oct 使用8进制
ios_base::hex 使用16进制
ios_base::floatfield ios_base::fixed 使用定点计数法(即一般计数法)
ios_base::scientific 使用科学计数法
ios_base::adjustfield ios_base::left 左对齐
ios_base::right 右对齐
ios_base::internal 符合或前缀左对齐，值右对齐

可以使用这种标识符进行定点输出，比如输出小数并且默认位数为小数点后两位


虚函数


虚函数很像Java中的抽象类的概念，在c++中，为基类创建一个虚析构函数是一种惯例

在类的继承中，原则之一（其实在其他程序里也是一样的），就是不要使用多余的程序，如果是在派生类中重新定义的方法，需要使用虚方法，否则就不需要使用

在基类方法的声明中使用virtual可以这个方法在基类和派生类中都是虚方法

如果使用指向对象的指针或引用来调用虚方法，程序将使用为对象类型定义的方法，称为动态联编，这样基类指针或引用可以指向派生类对象


int x = 2.5;
double * a = &x;//这是不允许的，c++中不允许将不同类型的数值赋给另一个类型的指针
long b = &x;//同理，引用也不可以

//在类的继承中，派生类的引用和指针就可以直接被基类赋值
Father x("cxy", 18);
Son * a = &x;
Son b = &x;

在定义基类中，那些要被派生类使用的方法要使用虚方法


使用虚函数的场景


构造函数：构造函数不能是虚函数，派生类的构造函数将使用基类的一个构造函数，这种顺序不同于继承机制，所以派生类不继承基类的构造函数

析构函数：析构函数一定要使用虚函数，在派生类过期时，一定要调用本类的析构函数

友元：友元不能是虚函数，友元不是类成员，只有成员才能使虚函数


访问控制-protected


private和protected的区别就在于基类的派生类，派生类的成员是可以直接访问基类保护成员的，但是不能访问私有成员

对于内部来说，保护类似于公有；对于外部来说，保护类似私有

不过，在一般情况下，我们都是采用私有访问，同时通过基类方法更改基类数据


抽象基类


在使用基类和派生类的时候，很多时候会存在细节上的不同，比如基类的方法或者数据派生类用不到

这个时候单独创建一个类又十分麻烦，在这种情况下我们使用纯虚函数


virtual double Area() const = 0;//这就是一个纯虚函数
void Move(int nx, int ny) = 0;//使用=0来创建纯虚函数
void BaseEllipse(int nx,int ny){//仍然可以对其进行定义
    
    x = nx;
    y = ny;
    
}
