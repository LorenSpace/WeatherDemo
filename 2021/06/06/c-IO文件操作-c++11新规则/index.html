c++ IO文件操作&c++11新规则

学习

文件操作基本理解


文件操作包含在头文件fstream中，打开文件之后编译器是可以自动关闭的，也可以使用显式方法close()来关闭文件（推荐）

对于一般的文本读写，可以直接使用>>和<<，并且对于某些需要char *类型的函数，如果使用了string类型，可以使用c_str()来将其转化成c语言风格的字符串

（使用is_open()查看是否成功打开文件）


#include <iostream>
#include <fstream>
#include <cstdlib>
using namespace std;

void CreateFile(string fileName);
void ReadFile(string fileName);
int main(){
    CreateFile("0x01.txt");
    ReadFile("0x01.txt");

    return 0;
}

void CreateFile(string fileName){
    ofstream fout(fileName.c_str());
    string str;
    if(!fout){
        cout << "Open Error" << endl;
        exit(1);
    }

    cin >> str;
    while(str != "#"){
        fout << str << endl;
        cin >> str;
    }
    fout.close();
}

void ReadFile(string fileName){
    ifstream fin(fileName.c_str());
    string str;
    if(!fin){
        cout << "Open Error" << endl;
        exit(1);
    }
    
    while(getline(fin, str, '\n')){
        cout << str << endl;
    }
    fin.close();
}

当然你可以使用用一个对象进行多个文件打开的操作，不过要记得显式关闭并且清除流中的残余

（使用clear()&close()）


命令行处理


c++有在能在命令行环境中运行的程序能后访问命令行参数的机制（这个参数和Java有点像）

具体表现为int main(int argc, char *argv[])，前者表示命令行参数的个数，后者变量则为一个指针（储存这命令行变量）。argv[0]就表示第一个命令行参数


文件模式精确操作


常量 含义
ios_base::in 打开文件，以便读取
ios_base::out 打开文件，以便写入
ios_base::ate 打开文件，并移到文件尾
ios_base::app 追加到文件尾（打开时不会重置文件内容）
ios_base::trunc 如果文件存在，则截短文件
ios_base::binary 二进制文件

ios_base::app模式只允许将数据添加到文件尾

ios_base::ate模式将指针放到文件尾


二进制文件


在C语言中就有接触过二进制文件，后缀尾dat

对于二进制文本的读写需要使用write()&read()，第一个参数表示目标字符，第二个参数则是字符的大小


#include <iostream>
#include <fstream>
#include <cstdlib>
#include <string>
using namespace std;

class Student{
    private:
        char number[20];
        char name[20];
        char sex[5];
        int scores;

    public:
        Student(char *nu = "", char *na = "", char *se = "", int s = 0);
        friend ostream& operator<<(ostream &out, const Student &s);
};

Student::Student(char *nu, char *na, char *se, int s){
    strcpy_s(number, nu);
    strcpy_s(name, na);
    strcpy_s(sex, se);
    scores = s;
}

ostream& operator<<(ostream &out, const Student &s){
    out << s.number << " " << s.name << " " << s.sex << " " << s.scores << endl;
    return out;
}

void CreateBiFile(string filename);
void ReadBiFile(string filename);
int main(){
    CreateBiFile("stu.dat");
    ReadBiFile("stu.dat");

    return 0;
}

void CreateBiFile(string filename){
    ofstream out(filename.c_str());
    if(!out){
        cerr << "File open error" << endl;
        exit(1);
    }

    Student stu[3] = {Student("B20032025", "陈鑫扬", "男", 99),
    Student("B20032024", "陈彦轩", "男", 98),
    Student("B20032023", "陈科廷", "男", 87)};
    out.write((char *)stu, sizeof(Student) * 3);
    out.close();
}

void ReadBiFile(string filename){
    Student stu[5];
	int i = 0;
    ifstream in(filename.c_str());
    if(!in){
        cerr << "File open error" << endl;
        exit(1);
    }

    while(!in.eof()){
        in.read((char *)&stu[i++], sizeof(Student));
    }
    for(i = 0; i < 3; i++){
        cout << stu[i];
    }
    in.close();
}

C++也有将指针进行位移的函数，seekg将输入指针移动到指定位置，seekp将输出指针移动到指定位置

包含两个参数，第一个是位移量，第二个是相对位置

相对位置包含三个，ios_base::beg（开始位置）；ios_base::cur（当前位置）；ios_base::end（末尾）


C++11新定义


新类型


c++11新增了类型long long&unsigned long long

char16_t&char32_t，以支持16位和32位的字符表示

c++11新增了右值(rvalue)的引用，使用&&表示，右值引用可关联到右值（可以出现在赋值的右边），但不能取地址

原本的左值(value)可以出现在赋值语句的左边，但是const修饰符出现之后就不行了，但是地址仍然可以使用


const int b = 101;//这样b就不能成为可使用的左值了，但是&b仍然可用
const int &rb = b;

int x = 10;
int y = 33;
int &&r1 = x + y;//这样我们规定的右值引用就可以放在赋值符左边

统一的初始化


c++11扩大了大括号的使用范围，可以添加等号（=），也可以不添加

虽然不能随便的类型转化，但是只要对应类型存在对应的值，就可以进行转化


int x = {23};
double y{2.5};
short sqrt{4.2, 5.6};

在能够使用大括号初始化后，就可以进行传入模板的操作了

c++中提供了initializer_list模板，可将其用作构造函数的参数，包含在头文件initializer_list


#include <iostream>
#include <initializer_list>
using namespace std;

double sum(initializer_list<double> li);
int main(){
    cout << sum({1.5, 23.7}) << endl;
    
    return 0;
}

double sum(initializer_list<double> li){
    double total = 0;
    for(auto i = li.begin(); i < li.end(); i++){//initializer_list<double> li::iterator i
        total += *i;
    }
    return total;
}
//多多使用auto自动声明

可以使用decltype强制让一个变量成为某种类型

decltype(x * n) p此时p就为x * n的类型，这可以在定义模板时起到作用（因为模板类只有到最后才知道类型）

此外，c++11还新增了一种返回值后置的写法。这种写法将返回的类型在后面指定，这样就可以使用decltype指定类型

auto sum(int x, double y)->double


template<typename T, typename U>
auto eff(T t, U u)->decltype(T * U){}

重命名


对于一些冗长的标识符，可以使用重命名的方式。以前提供的是typedef，c++11现提供了using=

这对于模板的具体化可以进行重命名，typedef则不行（使用新规则的可读性更强了）


#include <iostream>
#include <vector>
#include <string>
using namespace std;

// typedef vector<string>::iterator itType;
using itType = vector<string>::iterator;

int main(){
    vector<string>v{"cxy", "cyx", "ckt", "cmk"};
    for(itType i = v.begin(); i < v.end(); i++){
        cout << *i << " ";
    }
    cout << endl;

    return 0;
}

template<typename T>
using arr12 = array<T, 12>;

小细节优化


在枚举方法中，原本的枚举的变量是不能重名的。c++11新增可以使用struct或者class，这种枚举的使用很类似类和对象的不同方法的调用


enum class new1 {q, w, e, r};
enum struct new2 {q, w, e, r};
//在具体区分时，可以使用new1::q

原本为了避免使用模板时多个>导致编译混淆，需要使用空格来区分，现在不需要了

vector<list<int> > v1; vector<list<int>> v1;

初次之外，如果基类定义了多个重载的构造函数fin，在派生的定义中就可以使用using c1::fin;来时派生类访问所有的基类构造函数


Lambda函数


lambda的作用就是让程序员能使用一个匿名的函数（使用[]代替函数名），这在大篇幅代码的文件中比较实用

这里使用模板来举例，generate()接受一个区间（由前两个区间决定），最后一个参数是不接受任何参数的函数对象

使用cout_if()则是可以计算区间内有多少元素符合规定的要求


#include <iostream>
#include <vector>
#include <cstdlib>
#include <ctime>
#include <algorithm>
using namespace std;

int main(){
    vector<int> v(5);
    srand((unsigned int)(time(NULL)));
    generate(v.begin(), v.end(), rand);
    for_each(v.begin(), v.end(), [](int x){cout << x << " ";});//输出模板中的值
    cout << endl;
    cout << count_if(v.begin(), v.end(), [](int x){return x % 3 == 0;}) << endl;//判断模板中有多少值可以被3整除

    return 0;
}

但是这种新型的语法反而使可读性变差了，但是实际上，有些时候并非如此

如果一个文件中的代码量很多，对应表达式函数的代码往往是很难找到的，而且lambda也可以采用以下方式应用于多个要求使用同一个函数的代码参数


auto mod3 = [](int x){return x % 3 == 0;}
int count1 = count_if(v1.begin(), v1.end(), mod3);
int count2 = count_if(v2.begin(), v2.end(), mod3);
