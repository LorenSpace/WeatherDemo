深挖Field


主要方法


当然，如果不需要进行比较高级的操作，可以直接使用类名来进行反射操作

不过对于一些调用属性的操作依然需要类的对象

getType()： 获取属性声明时类型对象（返回class对象）

getGenericType() ： 返回属性声的Type类型

getName() ： 获取属性声明时名字

getAnnotations() ： 获得这个属性上所有的注释

getModifiers() ： 获取属性的修饰

isEnumConstant() ： 判断这个属性是否是枚举类

isSynthetic() ： 判断这个属性是否是 复合类

get(Object obj) ： 取得obj对象这个Field上的值

set(Object obj, Object value) ： 向obj对象的这个Field设置新值value

可以通过以上方法来对其进行的属性进行调用，在之前的例子中就已经有所使用

这些属性可以让反射编程快速调用一个类的各种具体信息，在Method中，方法也具体同理


小知识


AccessibleObject 类是 Field、Method 和 Constructor 对象的基类。它提供了将反射的对象标记为在使用时取消默认 Java 语言访问控制检查的能力。对于公共成员、默认（打包）访问成员、受保护成员和私有成员，在分别使用 Field、Method 或 Constructor 对象来设置或获得字段、调用方法，或者创建和初始化类的新实例的时候，会执行访问检查

这样就可以直接去访问私有成员了（x


set()&get()


set(Object obj, Object value);

可以通过这种方法，快速修改所调用field里面变量的值

public Object get(Object obj);

通过这种方法获取field里的属性值

这就很像类中固有的两种方法了，熟能生巧便能掌握全部（x


import java.lang.reflect.Field;

class Fish{
    private String name = "cxy";
    private int age = 5;
}

public class Field_set_get {
    public static void main(String[] args) throws Exception{
        Fish fish = new Fish();
        Field field = fish.getClass().getDeclaredField("name");
        field.setAccessible(true);//关闭Java访问规则，这样私有成员也能访问
        System.out.println(field.get(fish));
        field.set(fish, "fwh");
        System.out.println(field.get(fish));
    }
}

访问构造方法


getConstructors() 获取类中的公共方法

getConstructor(Class[] params) 获取类的特定构造方法,params参数指定构造方法的参数类型

getDeclaredConstructors() 获取类中所有的构造方法(public、protected、default、private)

getDeclaredConstructor(Class[] params) 获取类的特定构造方法,params参数指定构造方法的参数类型

（1）得到这个类的所有构造方法：如得到上面示例中Person类的所有构造方法

Constructor[] cons = Class.forName(“cn.itheima.Person”).getConstructors();

（2）获取某一个构造方法：

Constructor con=Person.class.getConstructor(String.class,int.class);

写多了你就会发现反射的语法调用都是类似的几种方法

要么就是直接连续调用，要么就是使用数组的东西获得整体


package Reflect;

import java.awt.*;
import java.lang.reflect.Constructor;

class Human{
    private String name;
    private String sex;
    private int age;
    private double scores;

    public Human(String name, String sex, int age, double scores){
        this.name = name;
        this.sex = sex;
        this.age = age;
        this.scores = scores;
    }

    public Human(){
        this("", "", 0, 0.0);
    }
}

public class Class_Constructor {
    public static void main(String[] args) {
        try {
            Class c = Class.forName("Reflect.Human");
            Human co = (Human) c.newInstance();
            Constructor c1 = co.getClass().getConstructor(null);
            System.out.println(c1);
            System.out.println();
            c1 = co.getClass().getConstructor(String.class, String.class, int.class, double.class);
            System.out.println(c1);
            System.out.println();

            Constructor[] c2 = co.getClass().getConstructors();
            for (Constructor constructor : c2){
                System.out.println(constructor);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Annotation


Annotation不影响程序的运行，但对编译器的警告工具会有一定作用（写在对应接口中）

定义的关键字为@interface，隐含了继承相关接口的意思


public @interface OneMemberAnnotation{
        String value();
}

String：成员类型，可用的类型有String、Class、primitive、enumerated、annotation，以及所有类型的数组

value：成员名称，如果类中只包含一个成员，名称通常为value

除此之外，还在ElementType中定义各种枚举常量进行注释（使用@target）

枚举常量 说明
ANNOTATION_TYPE 表示用于Annotation类型
TYPE 表示用于类、接口和枚举，以及Annotation类型
CONSTRUCTOR 表示用于构造方法
FIELD 表示用于成员变量和枚举变量
METHOD 表示用于方法
PARAMETER 表示用于参数
LOCAL_VARIABLE 表示用于局部变量
PACKAGE 表示用于包

也有RetentionPolicy类，表示Annotation的有效范围（使用@Retention）

枚举常量 说明
SOURCE 表示不编译Annotation到类文件中，有效范围最小
CLASS 表示编译Annotation盗垒文件中，但是在运行时不加载Annotation到JVM中
RUNTIME 表示在运行时加载Annotation到JVM中，有效范围最大 

public class EX_01 {
    @Ex_01_1(describe = "编号", type = int.class)//Java的Annotation提供了自定义注释的方法，可以对特定的代码块进行注释
    private int id;
    @Ex_01_1(describe = "姓名", type = String.class)
    private String name;

    @Ex_01_2("初始化构造方法")
    public EX_01(int id, String name){
        this.id = id;
        this.name = name;
    }
    @Ex_01_2()//调用注释中的默认值"默认构造方法"
    public EX_01(){
        this(0, "cxy");
    }
    @Ex_01_1(describe = "获取姓名", type = String.class)
    public String getName() {
        return name;
    }
    @Ex_01_1(describe = "设置姓名")//默认参数值为void.class
    public void setName(String name) {
        this.name = name;
    }
    @Ex_01_1(describe = "获取编号", type = int.class)
    public int getId() {
        return id;
    }
    @Ex_01_1(describe = "设置编号")
    public void setId(int id) {
        this.id = id;
    }
}

class Main{
    public static void main(String[] args) {
        EX_01 e = new EX_01();
        System.out.println(e.getId() + " " + e.getName());
        e.setId(20032025);
        e.setName("cmk");
        System.out.println(e.getId() + " " + e.getName());
    }
}

//以下是两个接口的代码
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

@Target({ElementType.FIELD, ElementType.METHOD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
public @interface Ex_01_1 {
    String describe();//定义一个没有默认值的String类型成员
    Class type() default void.class;//定义一个具有默认值的Class值成员
}

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

@Target(ElementType.CONSTRUCTOR)
@Retention(RetentionPolicy.RUNTIME)
public @interface Ex_01_2 {
    String value() default "默认构造方法";//定义一个具有默认值的String型成员
}
