对象与对象变量


想使用对象，就先构造对象，并指定其初始状态

Java中，使用构造器（constructor）来构造新实例

构造器的名字应该与类名相同所以Date类的构造器名为Date

想要构造一个Date对象，需要加上new


在构造函数中，可以直接使用this进行快速构建

import java.time.LocalDate;
import java.time.*;

//遍历月份的日历完成

public class LocalCalender {
    public static void main(String[] args) {
        LocalDate date = LocalDate.now();
        int month = date.getMonthValue();
        int today = date.getDayOfMonth();

        date = date.minusDays(today - 1);//设置一个月的开始，today-1设置了这个月的开始时间（因为是以数字计算的）
        DayOfWeek weekday = date.getDayOfWeek();//创建一个DayOfWeek对象,1 = Monday,..., Sunday = 7，可以表示这天是星期几
        int value = weekday.getValue();//将具体的值传给value

        System.out.println("Mon Tue Wed Thu Fri Sat Sun");
        for (int i = 1; i < value; i++){
            System.out.print("    ");
        }
        while (date.getMonthValue() == month){
            System.out.printf("%3d", date.getDayOfMonth());
            if (date.getDayOfMonth() == today){
                System.out.print("*");//如果是今天会加*
            }else{
                System.out.print(" ");
            }
            date = date.plusDays(1);
            if (date.getDayOfWeek().getValue() == 1){
                System.out.println();
            }
        }
        if (date.getDayOfWeek().getValue() !=1) {
            System.out.println();
        }
    }
}

now ()/of (int year, int month, int day) 得到当前时间的对象/得到给定日期的对象
getDayOfWeek 得到当前日期是星期几，作为DayOfWeek的实例返回。根据国际惯例，星期一为1
plusDays(int n)/minusDay(int n) 生成当前日期之后或之前n天的日期

即，getDayOfWeek()得到的是一周的枚举域

在一个类里的自定义


package Normal;

import java.time.LocalDate;

public class EmployeeTest {
    public static void main(String[] args) {
        Employee[] staff = new Employee[3];//定义三名Employee对象的数组

        staff[0]=new Employee("Jhon",7500,1999,5,1);//构造器通常通过new进行构造
        staff[1]=new Employee("Jho",8505,1999,6,2);//C++不需要进行new操作，但是Java构造器总是伴随着
        staff[2]=new Employee("Jh",9550,1999,7,3);

        for (Employee e:staff){//用foreach循环，对每个staff进行操作
            e.raiseSalary(7);
        }

        for (Employee e:staff){
            System.out.println(e.getName()+" "+e.getSalary()+" "+e.getHireDay());
        }
    }
}
class Employee{
    //private修饰的变量不能直接使用，只能确保自身类的使、
    private String name;
    private double salary;
    private LocalDate hireDay;

    public Employee(String n, double s, int year, int month, int day){//相当于构造器
        name = n;//不能在构造器中定义与实例域重名的局部变量
        salary = s;
        hireDay = LocalDate.of(year,month,day);
    }
//通过get一家将传入的参数输出
    public String getName() {
        return name;
    }

    public double getSalary() {
        return salary;
    }

    public LocalDate getHireDay() {
        return hireDay;
    }

    public void raiseSalary(double byPercent){
        double raise = salary * byPercent / 100;
        salary += raise;
    }//定义一个工资上调的方法
}

总结Java参数的使用情况

1.一个方法不能改变一个基本数据类型的参数

2.一个方法可以改变一个对象参数的状态

3.一个方法不能让对象参数引用一个新的对象

class Animal{

    {
        id = nextId;

        this.name = name;

        return name;

        return age;
    }

    public int getId() {

    }

调用构造器的处理步骤

1.所有的数据域被初始化为默认值

2.按照在类声明中出现的次序，依次执行所有域初始化语句和初始化块

3.如果第一个构造器调用了第二个构造器，则执行第二个构造器的主体

4.执行这个构造器的主体

类设计技巧总结


1.保证使用private，即私有化

2.一定不要对数据（局部变量）初始化

3.不要使用过多的基本类型

4.不是所有的与都需要独立的域访问器和域更改器

5.将职责过多的类进行分解

6.类名见名知意

7.优先使用不可变的类
